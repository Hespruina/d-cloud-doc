# 服务器运营规则

**适用范围**：所有加入本服务器的玩家  
**生效方式**：继续游玩服务器即视为接受本文件全部条款

## 0. 概述

为了确保每一位玩家都能获得稳定、公平且长期可发展的游戏体验，我们在此制定以下运行准则。本文件构成您与本服务器管理组之间的共识。您有责任阅读并理解其内容。**“未阅读规则”、“账号由他人操作”** 等理由，在发生违规行为时，不视为有效的免责条件。

::: danger 数据回溯权利说明
对于破坏游戏平衡的严重行为（例如：利用恶性漏洞、非法复制物品、恶意制造服务器崩溃），管理组保留对数据进行回溯的权利。一次严重违规，可能会导致您甚至全服玩家数十小时的建筑成果与资产被清除。

我们相信绝大多数玩家是遵守规则的。除少数“零容忍”行为外，管理组将在处理违规时参考玩家的态度与历史记录，并尽量提供改正机会。
:::

## 处罚等级说明

管理组采用五级处罚制度，视违规情节轻重、涉事玩家态度及历史记录综合裁定。各级别对应措施如下：

- **等级 1**：警告提醒
- **等级 2**：禁言 或 临时封禁 1 天
- **等级 3**：临时封禁 3–7 天，并视情况清除违规所得
- **等级 4**：临时封禁 30 天，清空个人资产或回溯数据
- **等级 5**：永久封禁，并在30天后清空全部数据，释放账号名。此类处罚的申诉将从严审核

零容忍行为将直接面临等级 5 处罚。后续条款中所提及的“3–5级处罚”“最高等级处罚”等均参照本表执行。

## 1. 聊天与社区规范

### 1.1 言论行为要求

- **遵守法律法规**：不得出现政治敏感、色情、赌博等内容。
- **相互尊重**：拒绝辱骂、引战，严禁任何形式的歧视性言论。
- **高效沟通**：避免直接索要，私聊管理员时，请陈述问题。

### 1.2 禁止扰乱秩序

- **聊天框**：禁止刷屏。
- **广告**：禁止发布其他服务器的宣传广告或链接。

## 2. 玩家财产与互动规则

### 2.1 尊重他人财产

当您被授予权限后：

- 不要偷窃箱子中的物品
- 不要恶意破坏他人圈地范围内的建筑
- 不要恶意击杀具有功能性的生物

::: tip
所有交互行为均有记录，请勿抱有侥幸心理。
:::

### 2.2 恶作剧的边界

我们理解玩家之间的玩笑行为，但请掌握分寸：

- 如果对方明确表示反感，请立即停止
- 请负责将环境恢复至原状

### 2.3 恶意行为

- **PvP 规则**：仅在双方明确同意的前提下允许 PvP。
- **诅咒装备**：禁止诱骗或强迫他人穿戴带有 `[绑定诅咒]` 的装备。一经举报，可给予处罚。

## 3. 公平游戏原则

### 3.1 禁止破坏公平性的行为

公平性是游戏体验的基础。任何破坏公平竞争的行为，一经确认，将面临 **4–5 级**的处罚等级。

**明确禁止：**

- 任何形式的作弊客户端
- 透视类材质包
- 用于 PvP 的自动连点器
- 自动寻路或行为脚本

**允许使用（需在合理范围内）：**

- **投影与打印**：允许使用，但请控制发包频率
- **挂机工具**：允许自动钓鱼类模组
- **灵魂出窍**：仅限用于建筑欣赏、机器排错或寻找截图角度

### 3.2 零容忍行为清单

以下行为触及底线，将面临**4~5级**处罚：

- **恶意利用漏洞**：发现服务器漏洞请立即报告管理组。隐瞒不报、私自利用者视为违规。
- **网络暴力与隐私侵犯**：严禁人肉搜索、开盒、曝光他人现实隐私，或进行人身安全威胁。
- **恶意破坏服务器性能**：禁止制造专门用于导致服务器卡顿或崩溃的装置。

## 4. 多账号使用政策

1. **基本原则**：每位玩家原则上允许拥有 1 个小号。
2. **红线行为**：非必要情况下，禁止小号操作服务器货币，禁止用于刷每日收购。
3. **违规处理**：违规使用小号将被定性为“恶意多开”。主号与小号将一并受到 **3–5 级**的处罚。

## 5. 聚落规则

### 5.1 建立聚落前的注意事项

- **睦邻友好**：在建立新基地前，请确认周边没有其他不熟悉玩家的基地。
- **合理圈地**：建议向外预留 3–4 个区块作为缓冲带。

::: info 小贴士：地图画制作
推荐在末地制作地图画——不占用主世界空间，也不易被意外破坏，何乐而不为呢
:::

### 5.2 纠纷处理

- 出现纠纷应首先尝试和平沟通。
- 禁止直接攻击或辱骂对方。

## 6. 经济秩序

- **禁止现实货币交易**：严禁使用人民币等现实货币交易游戏道具或账号。违规双方均会受罚。
- **市场公平**：禁止利用信息差，以欺诈性价格向新玩家出售低价值物品。

## 7. 红石机械规范

本服务器为非纯生电服。我们尊重并欢迎技术型玩家，但需要在服务器流畅度（TPS）与机器效率之间取得平衡。

### 7.1 禁止建造的机械类型

- 地狱门区块加载器
- 全物品分类系统
- 极高效率的刷怪塔（例如百万级猪人塔）

::: info 特殊申请
如果你真的想建造一个很有创意的`东西`，但苦于我们的规定，请毫不犹豫地联系管理团队们！我们会根据实际情况对您的`东西`重新进行审核并告知您是否可以进行建造。
:::

### 7.2 机械设计要求

- 任何可能造成大量堆叠（造成卡顿）的机器，都必须配备一键停机开关
- 长期运行的机器必须具备自动销毁或溢出处理机制

::: info 小贴士
强烈建议为所有机器配备总控开关
:::

### 7.3 运行规范

- **错峰运行**：在线人数高峰期，请自觉关闭高负载机器。若因机器运行导致其他玩家游戏体验严重下降，我们将迅速采取措施（严肃脸）。
- **实体限制**：禁止在一个区域内堆叠过多生物。

### 7.4 应急处理

若机器导致服务器崩溃，管理组可能会直接删除或重置该区块，且可能不提供任何补偿。

## 8. 申诉与反馈

1. 如果您认为处罚有误，或发现其他玩家存在违规行为，请通过私聊管理员进行申诉或举报。
2. **高效举报**：请务必提供截图、视频和具体时间点。
3. **理性交流**：请保持冷静与理性。威胁、辱骂管理组或其它越界行为可能导致处罚加重。
4. **管理裁量权**：管理组有权根据实际情况对违规行为进行处理。如果无法联系到涉事玩家，可处以最长永久的封禁。届时请通过申诉渠道联系管理组解决。
